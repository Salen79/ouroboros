# ROADMAP — Эволюция компании

> Стартовый бюджет: $1000 (единственная внешняя инвестиция)
> После $1000 — компания живёт на собственный доход (70/30 split, BIBLE.md P5)
> Инфраструктура: VPS ISHosting USA (New York) с Docker, CrewAI, CEO Dashboard
> См. docs/INFRASTRUCTURE.md для деталей

---

## === ПЕРВЫЙ ПРОДУКТ (стартовый капитал $1000) ===

### Phase 1: Discovery ($0-100)

**Цель:** Выбрать продукт, соответствующий миссии компании, и получить одобрение владельца.

**Ресурсы:** На сервере уже есть работающий Discovery Crew (CrewAI, 4 агента).
Можно использовать его или провести discovery самостоятельно через web search.
Результаты 4 прошлых прогонов доступны в ~/ai-company/outputs/.

**Задачи:**
- [ ] Изучить BIBLE.md — понять миссию и эволюционную цель
- [ ] Изучить docs/INFRASTRUCTURE.md — понять что доступно на сервере
- [ ] Изучить ~/ai-company/ — понять существующий код и возможности CrewAI
- [ ] Исследование рынка (web search, тренды, конкуренты)
- [ ] Миссионный фильтр: отсеять продукты не служащие осознанности
- [ ] Генерация и оценка идей по критериям из DISCOVERY_CRITERIA.md
- [ ] Product Brief для лучшей идеи (включая Mission Alignment)
- [ ] **CEO GATE 1:** отправить Brief владельцу через Telegram → одобрение

**Gate Criteria:** Product Brief одобрен владельцем.
**Бюджет:** до $100

---

### Phase 2: Foundation ($100-250)

**Цель:** Работающий MVP задеплоен и доступен по URL.

**Ресурсы:** VPS с Docker, PostgreSQL, Redis, ChromaDB.
Можно деплоить продукт прямо на сервере (порты 80/443 открыты)
или на внешние платформы (Vercel, Railway).

**Задачи:**
- [ ] Архитектура продукта (может использовать CrewAI agents или строить с нуля)
- [ ] Инициализация кодовой базы в company/
- [ ] Backend: core API endpoints
- [ ] Frontend: минимальный UI
- [ ] Тесты на критические пути
- [ ] CI через GitHub Actions
- [ ] Деплой (на VPS или внешнюю платформу)
- [ ] **CEO GATE 2:** живой URL → владелец тестирует

**Gate Criteria:** Продукт доступен по публичному URL, core функция работает.
**Бюджет:** до $150

---

### Phase 3: Product ($250-550)

**Цель:** Продукт полнофункционален и готов к пользователям.

**Задачи:**
- [ ] Реализовать все MVP-фичи из Product Brief
- [ ] Landing page с описанием, миссией и CTA
- [ ] Базовая аналитика (кто заходит, что делает)
- [ ] Документация для пользователей
- [ ] Error handling и edge cases
- [ ] **CEO GATE 3:** product review → ready for users?

**Gate Criteria:** Владелец готов показать продукт реальным людям.
**Бюджет:** до $300

---

### Phase 4: Users ($550-750)

**Цель:** Первые 10 реальных пользователей.

**Задачи:**
- [ ] SEO-оптимизация landing page
- [ ] Submit на Product Hunt / Hacker News / IndieHackers
- [ ] README с quick start guide
- [ ] Feedback mechanism (форма, email, или GitHub Issues)
- [ ] Итерация по фидбеку пользователей
- [ ] **CEO GATE 4:** есть пользователи? фидбек?

**Gate Criteria:** 10+ уникальных пользователей, собран фидбек.
**Бюджет:** до $200

---

### Phase 5: Monetization ($750-1000)

**Цель:** Первый доллар дохода. Переход к самофинансированию.

**Задачи:**
- [ ] Определить pricing model по данным usage
- [ ] Stripe/Lemon Squeezy интеграция
- [ ] Платный тариф или API pricing
- [ ] Оптимизация конверсии free → paid
- [ ] Настроить автоматический split: 70% → компания, 30% → создатель
- [ ] **CEO GATE 5:** есть revenue? компания self-sustaining?

**Gate Criteria:** Хотя бы $1 заработан. Модель revenue работает.
**Бюджет:** до $250 (остаток стартового капитала)

---

## === ПЕРЕХОД К САМОФИНАНСИРОВАНИЮ ===

После исчерпания $1000:

**Если revenue есть:**
- Компания продолжает на 70% от дохода
- 30% выплачивается создателю
- Переход к Phase 6

**Если revenue нет:**
- Компания останавливает активную разработку
- Отчёт владельцу: что сделано, почему не сработало, рекомендации
- Владелец решает: добавить бюджет или завершить

---

## === ПОСЛЕ ПЕРВОГО ПРОДУКТА (self-funded) ===

### Phase 6: Stabilize & Grow (funded by revenue)

**Цель:** Стабилизировать первый продукт, подготовиться к следующему.

- [ ] Автоматизация поддержки первого продукта
- [ ] Мониторинг: uptime, errors, user satisfaction
- [ ] Рост revenue до стабильного MRR
- [ ] Накопление резервного фонда (3 месяца операционных расходов)

### Phase 7+: Next Products (funded by revenue)

Повторить цикл: Discovery → Build → Ship → Grow → Monetize.
Каждый продукт быстрее предыдущего. Компания становится мудрее.

---

## Правила перехода между фазами

1. Нельзя перейти без прохождения CEO Gate
2. Если бюджет фазы исчерпан без Gate — отчёт владельцу с планом
3. Неизрасходованный бюджет переносится на следующую фазу
4. Владелец может перенаправить бюджет между фазами
5. Split 70/30 после самофинансирования — неизменен
