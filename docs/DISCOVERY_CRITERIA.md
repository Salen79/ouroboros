# Discovery Criteria — Какой продукт строить

> Компания непрерывно исследует потребности людей и создаёт продукты,
> которые служат жизни и росту осознанности (см. BIBLE.md).
> Этот документ определяет критерии для выбора каждого следующего продукта.

---

## Миссионный фильтр (проходит первым)

Прежде чем оценивать рынок и feasibility — каждая идея проходит тест:

**"Делает ли этот продукт жизнь людей более осознанной, свободной
или наполненной — хотя бы косвенно?"**

Примеры прохождения:
- Инструмент автоматизации, экономящий 2 часа/день → ДА (освобождает время)
- Аналитика, которая показывает скрытые паттерны → ДА (self-awareness)
- AI-ассистент для лучших решений → ДА (clarity)
- Казино-бот или генератор кликбейта → НЕТ (эксплуатация зависимостей)
- Ещё один ChatGPT-wrapper без добавленной ценности → НЕТ (не служит миссии)

Если идея не проходит миссионный фильтр — отбрасывается без дальнейшей оценки.

## Обязательные требования

### Scope
- Buildable в 3-7 дней
- Никакого hardware, никаких мобильных приложений
- Весь стек помещается на VPS (уже есть: Docker, PostgreSQL, Redis, ChromaDB)
  или на serverless-платформу

### Tech Stack
- Web-based: Python (FastAPI/Flask) или Node.js (Next.js) для backend
- Frontend: HTML/CSS/JS или React/Next.js
- Деплой: на текущем VPS, Vercel, Railway, или Fly.io
- БД: PostgreSQL уже доступен на сервере, SQLite для простых случаев

### Monetization
- Должен иметь чёткий путь к revenue (критично — после $1000 компания
  живёт только на собственный доход, BIBLE.md P5)
- Модели: SaaS подписка, API с usage-based pricing, freemium
- Целевой MRR: $100+ в течение 3 месяцев после запуска
- Никаких affiliate, рекламных или спекулятивных моделей

### Target Audience
- Developers, indie hackers, small businesses, фрилансеры,
  люди стремящиеся к осознанному образу жизни
- B2B или B2D (business-to-developer) предпочтительнее B2C
- Аудитория должна быть доступна через Product Hunt, Hacker News,
  Twitter/X, Reddit, IndieHackers

### Differentiation
- Должен использовать AI как core competency (не обёртка над ChatGPT)
- Решает конкретную боль, а не "ещё один AI-инструмент"
- Есть defensibility: данные, workflow интеграция, или network effects

## Критерии оценки (Score 1-10 по каждому)

| Критерий | Вес | Описание |
|----------|-----|----------|
| Mission Alignment | 20% | Насколько продукт служит осознанности и развитию жизни? |
| Feasibility | 20% | Можно ли построить за 3-7 дней с ограниченным бюджетом? |
| Monetization Clarity | 20% | Понятно ли как и за что люди будут платить? Быстро ли? |
| Market Size | 15% | Достаточно ли людей с этой проблемой? |
| Uniqueness | 15% | Есть ли прямые конкуренты? Чем мы лучше? |
| AI Leverage | 10% | Насколько AI — ядро продукта vs. просто feature? |

**Минимальный проходной балл: 7.0/10 (средневзвешенный)**
**Mission Alignment не может быть ниже 6/10** — иначе продукт отклоняется.

## Процесс Discovery

### Вариант A: Использовать CrewAI Discovery Crew
На сервере уже есть работающая Discovery crew (4 агента).
Можно запустить: `cd ~/ai-company && uv run python -m ai_company.main --crew discovery`
Стоимость одного прогона: ~$0.24-3.00

### Вариант B: Провести Discovery самостоятельно
Использовать собственный tool loop (web_search, read_file, write_file).

### Шаги Discovery (любой вариант)

1. **Research** (бюджет: до $50)
   - Web search: тренды, ниши, проблемы целевой аудитории
   - Анализ Product Hunt, Hacker News, IndieHackers за последний месяц
   - Изучить результаты предыдущих Discovery runs в ~/ai-company/outputs/
   - Особый фокус: продукты на стыке productivity + mindfulness + AI

2. **Ideation** (бюджет: до $20)
   - Сгенерировать 5-10 идей
   - Миссионный фильтр: отсеять не прошедшие
   - Оценить оставшиеся по критериям выше
   - Выбрать top-3

3. **Product Brief** (бюджет: до $30)
   - Для top-1 идеи написать полный Product Brief:
     - Problem Statement
     - Mission Alignment (как продукт служит осознанности)
     - Target Audience (конкретные персоны)
     - MVP Features (не более 5)
     - Tech Stack (что уже есть на сервере, что нужно добавить)
     - Monetization Model (помнить: после $1000 живём на revenue)
     - Competitive Landscape
     - 3-Month Revenue Target
     - Risks

4. **CEO Gate**
   - Отправить Product Brief владельцу через Telegram
   - Ждать: Approve / Request Revision / Reject
   - При Reject — вернуться к шагу 2 с другой идеей

## Для второго и последующих продуктов

Когда первый продукт стабилизирован и приносит revenue:
- Повторить процесс Discovery с нуля
- Учитывать опыт первого продукта (что сработало, что нет)
- Учитывать синергию с существующим портфелем
- Бюджет Discovery финансируется из 70% revenue (BIBLE.md P5)

## Примеры хороших продуктов (для ориентира)

- API-сервис который решает конкретную задачу разработчика
- SaaS-инструмент для автоматизации рутины, освобождающий время
- AI-powered анализатор, помогающий видеть паттерны
- Developer tool с CLI + web dashboard
- Micro-SaaS для осознанного управления временем/вниманием/энергией
- Инструмент для фрилансеров, убирающий administrative burden

## Примеры плохих продуктов (НЕ строить)

- Ещё один AI-чатбот
- Обёртка над GPT/Claude API без добавленной ценности
- Социальная сеть или marketplace (слишком сложно для MVP)
- Продукт требующий большую базу пользователей для ценности
- Anything requiring regulatory compliance (fintech, health)
- Продукты, эксплуатирующие FOMO, зависимости или бессознательное потребление
- Генераторы спама, кликбейта или манипулятивного контента
